## 📋 接下来你需要做的事

### 方案 1：使用 Railway 部署（最简单，推荐！）

#### 第 1 步：上传代码到 GitHub

1. 打开浏览器访问：https://github.com
2. 登录你的 GitHub 账号（没有就注册一个）
3. 点击右上角 **+** → **New repository**
4. 填写：
   - Repository name: `pms-system`
   - 选择 **Public**（公开）
   - 不要勾选 "Initialize this repository with README"
5. 点击 **Create repository**

6. 复制页面显示的推送命令，在 PowerShell 中执行：
```powershell
cd D:\qt\pms-system
git remote add origin https://github.com/你的用户名/pms-system.git
git branch -M main
git push -u origin main
```

#### 第 2 步：部署到 Railway

1. 访问：https://railway.app
2. 点击 **Start a New Project**
3. 选择 **Deploy from GitHub repo**
4. 选择你的 `pms-system` 仓库
5. Railway 会自动识别并部署

#### 第 3 步：配置后端服务

1. 在 Railway 项目页面，点击 **+ New** → **Empty Service**
2. 在 Settings 中配置：
   - **Root Directory**: `backend`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
3. 点击 **Deploy**

#### 第 4 步：配置前端服务

1. 再次点击 **+ New** → **Empty Service**
2. 在 Settings 中配置：
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Start Command**: `npm run preview -- --host 0.0.0.0 --port $PORT`
3. 点击 **Deploy**

#### 第 5 步：获取你的公网 URL

部署完成后，Railway 会给你两个 URL：
- 后端 API：`https://pms-backend-production.up.railway.app`
- 前端页面：`https://pms-frontend-production.up.railway.app`

**🎉 现在任何人都可以访问你的前端 URL 来使用系统了！**

### 方案 2：使用 Vercel + Railway（最佳性能）

#### 前端部署到 Vercel

1. 访问：https://vercel.com
2. 用 GitHub 账号登录
3. 点击 **Add New...** → **Project**
4. 选择你的 `pms-system` 仓库
5. 配置：
   - **Framework Preset**: Vite
   - **Root Directory**: `frontend`
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
6. 点击 **Deploy**

#### 后端部署到 Railway

同上方案 1 的第 3 步。

### 方案 3：临时演示用 ngrok（5 分钟搞定）

如果你只是想临时给别人演示一下：

#### 第 1 步：启动本地服务

打开两个 PowerShell 窗口：

**窗口 1 - 后端：**
```powershell
cd D:\qt\pms-system\backend
python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

**窗口 2 - 前端：**
```powershell
cd D:\qt\pms-system\frontend
npm run dev
```

#### 第 2 步：安装并运行 ngrok

1. 访问：https://ngrok.com/download
2. 下载 Windows 版本
3. 解压后，在两个新窗口分别运行：

```powershell
# 窗口 A - 暴露后端
ngrok http 8000

# 窗口 B - 暴露前端
ngrok http 3000
```

4. ngrok 会给你两个临时 URL，例如：
   - `https://abc123.ngrok.io` (后端)
   - `https://def456.ngrok.io` (前端)

**⚠️ 注意**: ngrok 免费版每次重启 URL 会变，只适合临时演示。

## 🔧 重要：修改前端 API 地址

部署后，前端需要知道后端的公网地址。

编辑文件：`frontend/src/api/request.js`

找到这一行：
```javascript
const BASE_URL = 'http://localhost:8000'
```

改为你的后端公网地址：
```javascript
const BASE_URL = 'https://你的后端地址.up.railway.app'
```

然后重新提交代码：
```powershell
cd D:\qt\pms-system
git add .
git commit -m "Update API base URL for production"
git push
```

## 📊 部署方案对比

| 方案 | 优点 | 缺点 | 适合场景 |
|------|------|------|----------|
| **Railway** | 免费、简单、自动 HTTPS | 免费层有额度限制 | 长期展示 |
| **Vercel + Railway** | 前端 CDN 加速、性能最好 | 需要配置两个平台 | 生产环境 |
| **ngrok** | 5 分钟搞定、无需注册 | URL 会变、不稳定 | 临时演示 |
